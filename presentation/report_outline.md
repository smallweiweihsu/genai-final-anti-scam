# Report Outline

## 1. 專案動機

- 詐騙訊息常利用急迫感、權威感與資訊不對稱。
- 一般使用者需要快速、白話的風險提醒。

## 2. 專案目標

- 建立可互動的防詐分析 demo。
- 整合文字案例、圖片案例、模型輸出與安全建議。

## 3. 系統設計

- 前端單頁網站。
- 文字分析為第一版核心。
- 圖片上傳先做 mock，未來可延伸 OCR 或多模態分析。

## 4. 資料集設計

- text_cases.csv 欄位與標註規則。
- image_cases.csv 欄位與標註規則。
- 高風險、低風險、中風險比例。

## 5. 模型與 API

- 參考 LoRA phishing email detection。
- 使用 Experiment B 輸出形式作為第一版 schema。

## 6. Demo 展示

- 假物流補繳運費。
- 假銀行帳戶驗證。
- 正常 IT 維護通知。
- 中風險模糊案例。

## 7. 限制與未來工作

- 第一版圖片不做真實模型判斷。
- 未來可加入 OCR、Gemini 多模態或更完整的中文本土資料。
