# 真假訊息守門員：生成式 AI 防詐內容分析器

本專案是一個期末專題用的單頁互動式網站，讓使用者貼上可疑文字或上傳可疑圖片後，取得詐騙風險分析、可疑特徵、詐騙話術、個資與付款風險，以及一般使用者能理解的安全建議。

第一版展示以文字分析為主，圖片上傳先提供圖片預覽與 mock 分析結果。模型串接完成前，前端會先讀取 `web/mockData.js` 的範例資料。

## 專案目標

- 判斷訊息是否可能為 phishing / scam。
- 顯示風險等級與信心分數。
- 說明可疑特徵、詐騙話術、個資風險與付款風險。
- 提供清楚、可執行的安全建議。
- 整理可供模型負責人、網站負責人與期末展示共同使用的資料格式。

## 分工

- A：資料與案例整理，包含文字案例、圖片案例、標註規則與 demo 案例。
- B：模型與 API schema，參考 LLaMA-Factory + LoRA phishing email detection 經驗。
- C：網站、影片與期末展示整合。

## 第一版資料策略

- 文字案例先完成 100 筆，再擴充到 300 筆。
- 正常 / 低風險案例至少 40%。
- 高風險 phishing / scam 約 50%。
- 中風險 / 模糊案例約 10%。
- 期末公開 demo 優先使用自製泛稱案例，避免真實品牌、個資與授權問題。

## 網站第一版

網站位於 `web/`：

- `index.html`：單頁介面。
- `style.css`：視覺樣式。
- `script.js`：互動邏輯。
- `mockData.js`：前端暫用分析資料。

## 安全與授權限制

不要上傳 API key、`.env`、大型模型檔、真實個資截圖、授權不明圖片、大量 raw images、真實釣魚網站內容或可疑附件。

若資料來源授權不明，只記錄來源到 `dataset/sources.md`，不要重新散布原始內容。
