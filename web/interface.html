Interface Design

介面設計

介面不是只顯示模型分類，而是把模型輸出轉換成可理解、可行動的安全提醒。

文字 / 圖片雙模式輸入

使用 tab 降低視覺負擔，讓使用者一次只處理一種輸入。

risk_score 與 confidence_score 分離

risk_score 表示危險程度，confidence_score 表示判斷信心，避免誤導。

可疑特徵 tags

把 indicators 變成易掃讀的標籤，例如急迫語氣、要求登入、可疑連結。

詐騙話術 tags

呈現社交工程手法，例如製造急迫感、冒充官方、付款誘導。

個資 / 付款風險

分開提醒使用者可能失去什麼資料或金錢。

安全行動 checklist

把建議轉成清楚步驟，例如不要點連結、改走官方 App。

如何串接模型 API

目前 prototype：

const result = analyzeEmail(messageInput.value);
renderResult(result);

未來 API：

const response = await fetch("/api/analyze-email", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ text: messageInput.value })
});
const result = await response.json();
renderResult(result);

只要模型回傳符合 docs/api_schema.md 的 JSON，前端就可以直接顯示。
